Clivia Generator

Este es un programa que te permite responder 10 preguntas de genero randomico, por cada respuesta correcta se obtiene 10 puntos, 0 puntos por respuesta incorrecta, al finalizar se te permitira guardar el nombre del jugador y la opcion de visualizar los puntajes en orden de merito. Este programa realiza solicitudes efectivas y obtiene respuestas a través de la red usando Ruby, trabajando con API (TRIVIA) para manejar, usar y guardar la información de manera eficiente en un archivo JSON

En esta parte de ruby trabajamos con la gema de ruby json (require "json") para el manejo de archivos de formato JSON , httparty (require "httparty") para conexion URL de internet, terminal table (require "terminal-table") para impresion en formato de tabla.

El API a consumir en este proyecto es el OPEN TRIVIA API que se encuentra en AQUI

Pasos para ejecutar el funcionamiento del proyecto

Paso Nro 1: Clonar el repositorio

Realizamos lo siguiente:

  • Create una carpeta y en alli abre un teminal
  • En la terminal ejecutamos el comando git clone git@github.com:cverah/ruby_Generador_CLIvia.git
  • luego de ello ingresamos a la carpeta clonada (se tiene que visualizar los archivos clonados)
Paso Nro 2: Ejecutar Bundle install

Una vez terminado el paso 1 realizamos la instalacion de bundle install para que se puedan descargar las gemas del proyecto en este caso el json, terminal-table y httparty

  • en la terminal ejecutamos el comando bundle install
  • Esperamos a que se descarguen las gemas el tiempo va depender de la velocidad de descarga de su internet
Paso Nro 3: Ejecutar el proyecto

Una vez terminado el paso 2 ejecutamos el proyecto para ello hacemos lo siguiente:

  • Ejecutamos el archivo main.rb con el comando ruby main.rb
  • Ya dentro del programa escribimos random y el sistema mostrara preguntas randomicas que tendran que responder por el indice de las respuestas
Paso Nro 4: Opcion Score

Ejecutamos lo siguiente:

  • Al finalizar las 10 preguntas el usuario tiene la opcion de guardar su puntaje con su nombre y en caso no introduzca su nombre el programa lo guardara como Anonimo
  • Se cuenta con la opcion score que muestra el los puntajes de cada usuario ordenados de manera descendente
Paso Nro 4: Guardar en archivo personalizado

Se cuenta con una opcion adicional para que se pueda guardar los scores en un archivo dado por el usuario

  • Salir del programa con la opcion exit
  • Ejecutamos nuevamente el programa pero esta dandole con ruby main.rb custom_scores.json en donde el nombre custom_scores.json es el archivo json en donde se guardaran los puntajes (se le puede poner el nombre que desee al archivo json pero siempre con el formato.json al final)
  • Para mas informacion sobre el proyecto ingresa a README.md, asimismo recordar el modelo de Happy paths (buen usuario) en los programas

Como fue construido

En nuestro equipo de trabajo fue construido de la sgte manera

Conectando a la terminal

En un prinicipio el grupo de trabajo trata de entender lo que quiere que el programa realice, una vez terminado se reparte las tareas iniciales y se crea los issues en github a cada integrante del grupo, una vez terminada se comparte la tara realizada y pueda ser aprobada por el equipo de trabajo

Comenzar a codear

Una vez que cada integrante tiene su tarea asignada, el integrante crea su branch(rama) en git y luego comienza a realizar el codigo en el lenguaje de programacion ruby que satisfaga la tarea en mencion y tratando de optimizar dicho codigo (para ello en ruby usamos el RUBOCOP)

Creacion de commit en git a github

Una vez terminado el codigo y siendo esta aprobado por el equipo de trabajo se realiza un push dentro de la rama donde trabajo a la rama de prueba (en la mayor parte con nombre develop) se resuelven los conflictos en caso haya y nuevamente regresar a la asignacion de tareas hasta completar el proyecto.Una vez finalizado terminado y realizado prueba se migra la rama de prueba (develop) a la rama principal que es main
